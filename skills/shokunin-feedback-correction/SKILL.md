# Shokunin Feedback Correction

## Overview

Use this skill when Shokunin Review produced an incorrect or unhelpful result.

## Feedback Types

- false positive
- missed issue
- score too high
- score too low
- not actionable
- too generic
- overengineered
- hallucination
- wrong document type
- wrong audience

## Process

1. Identify the incorrect output
2. Capture user correction
3. Save feedback log
4. Create eval candidate
5. Do not automatically mutate scoring or validators
