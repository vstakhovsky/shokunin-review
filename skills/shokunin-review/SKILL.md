# Shokunin Review

## Overview

Use this skill to review a document before a human reviewer does.

It checks whether the document is clear, evidence-backed, decision-ready, and actionable.

## When to Use

Use for:

- PRDs
- RFCs / technical design docs
- Experiment plans
- Product strategy docs

## Review Process

1. Parse artifact
2. Classify document type
3. Build review split-lens review
5. Apply quality gates
6. Calculate readiness score
7. Generate report
8. Suggest next actions
9. Support feedback / correction loop

## Core Principles

- Review-first, not rewrite-first
- Find problems, do not make them up
- Penalize unsupported claims
- Prefer minimal useful improvements
- Make scoring traceable
